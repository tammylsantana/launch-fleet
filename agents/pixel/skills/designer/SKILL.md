# Designer Skill

Create logos, interfaces, and visual systems. Applies visual hierarchy, branding, and usability principles to every design decision.

## Visual Hierarchy
1. **Size + weight** guide attention — one focal point per screen
2. **Whitespace** groups and elevates elements (not filler)
3. **Alignment** via grids creates intentional order
4. Design **empty states, errors, and loading** screens FIRST

## Color Theory (60-30-10 Rule)
- **60%** dominant (background, large areas)
- **30%** secondary (cards, sections, navigation)
- **10%** accent (CTAs, links, active states)
- Warm colors advance, cool colors recede — use for depth
- **4.5:1 contrast ratio** minimum (WCAG AA)
- Test with colorblind simulators (~8% of men affected)

## Typography Rules
- **Two typefaces maximum** (one heading, one body)
- Body text: **minimum 16px**
- Line height: **1.4-1.6**
- Line length: **45-75 characters** for readability
- Scale with clear hierarchy jumps

## Spacing (8px Grid)
- All spacing in multiples of 8: 8, 16, 24, 32, 40, 48...
- **Rule of thirds** for dynamic layouts
- Touch targets: **44px minimum**
- Consistent padding within component families

## Logo Design
- Must work as a **16px favicon**
- Recognizable in **single color** (no gradient dependence)
- Test on **both light and dark** backgrounds
- Maximum **3-5 brand colors**
- Simple enough for a child to draw from memory

## Icon Design
- **Consistent stroke weights** across the set
- Clear meaning **without labels** when possible
- Style must match brand personality (playful vs corporate)
- Provide all states: default, active, disabled

## Design Systems
- Think **components, not pages**
- Use tokens for colors, spacing, typography
- Document **when** to use components, not just how they look
- Include all states: hover, active, disabled, error, loading

## Responsive Design
- **Mobile first**, then scale up
- Specs in **relative units** (rem, %, vw)
- Touch targets: **44px minimum**
- Test at 320px, 375px, 414px, 768px, 1024px, 1440px

## Design Handoff
- Present designs **in context** (mockups on devices)
- Include **all states** (hover, active, disabled, error, loading)
- Provide spacing specs and color tokens
- Export assets at 1x, 2x, 3x for iOS
