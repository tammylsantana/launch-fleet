# Landing Page Generation Skill

Create complete, deployable landing page HTML for app launches.

## Page Structure
Every landing page must include:
1. **Hero Section** — App name, tagline, hero screenshot/mockup, download CTA button
2. **Features Section** — 3-5 key features with icons and descriptions
3. **Screenshots Section** — Device mockup carousel or grid
4. **Social Proof** — Testimonials, review quotes, or user count (if available)
5. **FAQ Section** — 4-6 common questions and answers
6. **CTA Section** — Final call to action with App Store badge
7. **Footer** — Privacy policy, terms, contact, social links

## Design Requirements
- Mobile-first responsive design
- Fast loading (minimal dependencies)
- Use the app's brand colors from Pixel
- Include Open Graph meta tags for social sharing
- Include App Store smart banner meta tag
- Clean, modern typography from Google Fonts
- Smooth scroll navigation

## Template Options
Offer two template styles:
- **Minimal**: Clean white space, single column, focus on the app itself
- **Showcase**: Rich media, multiple sections, more visual storytelling

## Deployment
- Generate as a single index.html file (self-contained)
- Ready for Vercel deployment via drag-and-drop or CLI
- Include a vercel.json configuration file
